### 0-2. 최신월 식별 — **새로 만들지 말 것. 기존 값 2개로 충분** [확인]

이미 존재하는 것:
- `DealTypeSummary.current_ym` — `lib/analyzer.ts:273-274`에서 `avg !== null`인 가장 최근 달로 계산되어 `:308`로 내려온다. `SummaryCards`는 **이미 이 값을 받아 `(2026.07)` 라벨을 렌더 중** (`SummaryCards.tsx:64`).
- `getMonthKey(offset)` — `lib/analyzer.ts:195` `export`. 내부적으로 `lib/kst.ts getKstYm` 사용이라 **KST 기준이 보장**되고, `getYmList`(수집 범위)와 같은 기준임이 주석에 명시돼 있다 (`:184-194`).

**판정 규칙 (개발팀 그대로 사용):**

```ts
// 새 로직이 아니라 기존 두 값의 비교. 표시값에 영향 없음.
const isPartialMonth =
  summary?.current_ym != null && summary.current_ym === getMonthKey(0);
```

이 규칙을 쓰는 이유:
1. **KST 안전.** 로컬 `new Date().getMonth()`를 새로 쓰면 서버/클라 타임존이 갈린다. `getMonthKey(0)`은 이미 KST로 통일돼 있다 [확인].
2. **거래 없는 지역에서 오공시를 막는다.** 7월 거래가 0건이면 `current_ym`은 `202606`이 된다. 이때 "이 달은 신고 기한 중"이라고 붙이면 **거짓 공시**다. 위 규칙이면 배지가 뜨지 않는다.
3. **차트에서 엉뚱한 구간에 점선이 가는 사고를 막는다.** `MonthlyChart`는 `spanGaps: true`라 최신월 avg가 null이어도 축에는 남는다 (`MonthlyChart.tsx:29-31`). `current_ym === getMonthKey(0)`가 참일 때만 `current_ym`이 **곧 마지막 인덱스**임이 보장된다.

**[확인] `monthly` 키는 항상 13개월 전부 존재한다** — `app/api/data/route.ts:75-76` `getYmList(months)`로 키를 먼저 깔고, 거래 0건 월도 `avg:null, count:0`으로 채워진다 (`analyzer.ts:245-252`). 따라서 "마지막 키 = 이번 달"이 성립하지만, 위 1·2번 이유로 **인덱스 위치가 아니라 `getMonthKey(0)` 비교**를 쓴다.

**한계 (범위 밖, 기획팀 후속 판단 요청):** 신고 기한이 계약일+30일이므로 엄밀히는 **직전월도 아직 열려 있다**(6월 28일 계약 → 기한 7월 28일). 이번 라운드는 승인 범위 문구인 "최신월"에 맞춰 당월만 표시한다. 직전월까지 확대할지는 별건 판단 항목으로 올린다.

### 0-3. 모바일 레이아웃 파손 여부 — **파손 없음** [확인 + 일부 추정]

`SummaryCards.tsx:60` 그리드: `grid-cols-1 gap-2 md:grid-cols-[repeat(auto-fit,minmax(200px,1fr))]`
- **모바일**: 1열이라 첫 카드가 화면 폭 전체. 배지 삽입 여유 충분.
- **데스크톱**: 카드 최소폭 200px − `p-5`(좌우 40px) = **콘텐츠 폭 160px**. 현재 라벨 `현재 평균 매매가 (2026.07)`만으로도 `text-xs`(12px) 기준 이미 2줄로 접힐 가능성이 높다 **[추정: 실측 렌더 미확인]**. → 배지를 같은 줄에 **강제로 붙이지 말고 `flex flex-wrap`**으로 자연 줄바꿈시킨다(§2-2). 그리드 기본 `align-items: stretch`라 한 카드가 한 줄 늘어나면 같은 행 카드가 함께 늘어날 뿐 **레이아웃이 깨지지 않는다**.
- **금지**: 배지에 `whitespace-nowrap` + 라벨 `truncate` 조합. 좁은 폭에서 월 라벨이 잘려 `(2026.0…)`이 되면 공시가 훼손된다.

`MonthlyChart.tsx:82` `h-[220px] md:h-[300px] w-full` — 차트 캔버스는 고정 높이 컨테이너 안이고, 캡션은 **컨테이너 바깥 아래**에 두므로 캔버스 크기에 영향 없다.

## 1. 문구 확정안

### 1-1. 배지: **`신고 진행 중`** (기획 A안 채택)

- 기획팀 비권장 `집계 중` 배제 근거 그대로 승계: 집계 주체가 우리로 읽혀 "로딩 중"으로 오인.
- `잠정치`(B안)도 검토했으나 **주체 모호성이 `집계 중`과 같은 방향의 문제**다("누구의 잠정치?" → 우리 계산이 덜 됐다는 오독). 기각.
- `신고 기한 미도래`(C안)는 배지로 너무 길다. 본문에서 같은 내용을 서술하므로 배지에 담을 필요가 없다.

> **참고 — 이번 범위 밖 발견 [확인]:** `SummaryCards.tsx:38`이 변동률 null일 때 이미 **`수집 중`**을 렌더한다. 기획팀이 `집계 중`을 기각한 것과 **정확히 같은 주체 오독 문제**가 이미 화면에 있다. 새 배지 `신고 진행 중`과 한 화면에 나란히 뜨면 사용자가 두 문구를 같은 뜻으로 읽을 수 있다. 승인 범위 밖이라 **이번 라운드에서 건드리지 않는다.** 별건 등록을 마스터께 요청한다.

### 1-2. 본문 (SummaryCards 하단) — **기획 A안 본문을 수정해 채택**

> 부동산 거래 신고는 계약일로부터 30일 이내에 하도록 되어 있습니다. 표시된 가장 최근 달(**2026.07**)은 이 신고 기한이 지나지 않은 기간이어서, 이후 다시 조회하면 이 달의 평균가가 달라질 수 있습니다. 위 변동률과 차액도 모두 이 달의 평균가를 기준으로 계산된 값입니다.

**기획 A안 원문에서 무엇을 왜 고쳤는지:**

| 기획 A안 원문 | 확정안 | 사유 |
|---|---|---|
| "신고 기한이 지나지 않아 **일부 거래가 아직 반영되지 않은 상태이며**" | "이 신고 기한이 지나지 않은 기간이어서" | 마스터 제약 2가 **"아직 덜 반영됐습니다" 류를 명시적으로 금지**했다. 원문은 이 표현을 사실상 그대로 쓴다. "반영 안 된 게 있다"는 곧 "채워지면 늘어난다"로 읽히고, 해제(취소)거래 필터를 (a)로 정하면 건수·평균가가 **내려가므로** 그 시점에 거짓 공시가 된다. → **원인(기한)만 서술하고 결과의 부호는 말하지 않는다.** |
| (없음) | "위 변동률과 차액도 모두 이 달의 평균가를 기준으로" | 기획 §2의 핵심 지적(부분월 값 1개가 카드 5장을 지배)이 원문 문구에 반영돼 있지 않았다. 배지는 첫 카드에만 붙는데 영향은 5장 전부에 미치므로, **배지가 닿지 않는 4장을 본문이 덮는다.** |
| "값이 달라질 수 있습니다" | "평균가가 달라질 수 있습니다" | 대상 명시. 원인은 열어둔 채 끝나는 구조는 그대로 유지(기획 §5-1 요구사항 — 자연키 확장으로 값이 바뀌어도 문구를 다시 쓸 필요 없음). |

**방향 중립성 자체 검증** (사용자 판단 대기 3건 전부에 대해):
- 해제거래 필터 (a)/(b)/(c) 무엇이 되든 → "달라질 수 있습니다"만 있으므로 참.
- 자연키 확장 (A)/(B) 무엇이 되든 → 원인을 신고 지연으로 못박지 않고 "이후 다시 조회하면"으로 끝나므로 참.
- 전세 모집단 통일 (A)/(B)/(C) 무엇이 되든 → 이 문단은 모집단을 언급하지 않으므로 무관.

**YMYL 금지 표현 (개발팀 전달용, 기획 §4 승계 + 추가):** "확정되면 오릅니다" / "실제 거래는 더 많습니다" / "저평가" / "아직 반영 안 된 거래가 있으니 참고하세요" / "정확한 시세는 다음 달에" — **전부 금지.** 데이터의 상태만 서술하고 그 함의는 말하지 않는다.

```tsx
  {dealType === "전세" && (
    <p className="text-xs text-gray-500 leading-relaxed"> …기존 문장 그대로… </p>
  )}
</div>
```

- `space-y-2` 래퍼 — 두 문단이 동시에 뜰 때(전세 탭 + 당월 데이터 존재) 붙어서 한 덩어리로 보이지 않게. 기존 `<p>`에는 마진이 없었으므로 래퍼로만 처리한다.
- 클래스 `text-xs text-gray-500 leading-relaxed`는 **기존 공시문과 완전히 동일**(`SummaryCards.tsx:88`, `DealsTable.tsx:85`, `AptDetailModal` 하단 공시) [확인]. 새 타이포 스케일을 만들지 않는다.
- `ymLabel` = `2026.07` 형태. 라벨 포맷 로직이 `L64`와 중복되므로 컴포넌트 상단에서 한 번만 계산해 양쪽이 공유하게 한다(값 변경 아님, 문자열 조립만).

### 2-3. 하지 말 것

- ChangeCard 4장에 배지를 각각 붙이지 말 것 → 배지 5개는 소음이고, 본문 마지막 문장이 이미 4장을 덮는다.
- 카드 배경색·테두리를 바꾸지 말 것 → 경고색으로 읽히면 YMYL상 "이 수치를 믿지 말라"는 과잉 신호가 된다. 우리는 상태만 알린다.

### 3-2. 인덱스 산출 + 데이터셋 변경

```tsx
const sortedYms = Object.keys(monthly ?? {}).sort();          // 기존 L29 그대로
const isPartial = currentYm !== null && currentYm === getMonthKey(0);
const lastIdx = isPartial ? sortedYms.indexOf(currentYm!) : -1;   // 없으면 -1
```

데이터셋에 **추가만** 한다(`MonthlyChart.tsx:36-44`의 기존 속성은 전부 유지):

```tsx
{
  label: …,                    // 기존 유지
  data: values,                // ★ 기존 유지 — 값 변경 없음
  borderColor: "#3f51b5",      // 기존 유지
  backgroundColor: "rgba(63, 81, 181, 0.15)",
  spanGaps: true,
  tension: 0.3,

  // 마지막 구간만 점선. chart.js 4.5.1 ScriptableLineSegmentContext 지원 확인함.
  // undefined를 반환하면 기본(실선)으로 폴백된다.
  segment: {
    borderDash: (ctx) => (ctx.p1DataIndex === lastIdx ? [6, 4] : undefined),
  },

  // 점 강조: 마지막 점만 크게 + 흰 속 채움(= "아직 열린 값" 관례).
  pointRadius: sortedYms.map((_, i) => (i === lastIdx ? 5 : 3)),
  pointBackgroundColor: sortedYms.map((_, i) =>
    i === lastIdx ? "#ffffff" : "rgba(63, 81, 181, 0.15)"
  ),
  pointBorderColor: "#3f51b5",
  pointBorderWidth: sortedYms.map((_, i) => (i === lastIdx ? 2 : 1)),
}
```

- `pointRadius: 3`(기존 `L43`)을 배열로 바꾼 것이라 **비최신월 점의 외형은 그대로**다.
- `pointBackgroundColor` 기본값을 `rgba(63,81,181,0.15)`로 명시하는 이유: 미지정 시 점 채움색이 데이터셋 `backgroundColor`를 따르므로 현재 렌더와 동일한 값을 그대로 적어 **외형 회귀를 막는다** [확인: `backgroundColor`가 `L40`에 그 값으로 지정돼 있음].
- 점선 패턴 `[6, 4]`: 220px 높이 모바일에서도 구분되는 최소 크기 **[추정 — 개발팀 구현 후 §5 검수에서 실측 조정 가능]**.
- **선 색을 흐리게(투명도↓) 하지 않는다.** 회색조로 죽이면 "이 값은 신뢰도가 낮다"는 판단을 우리가 대신 내리는 셈이라 YMYL상 부적절하고, 저대비로 접근성도 떨어진다. **모양(점선)으로만 구분한다.**

**엣지 케이스 (개발팀 필수 처리):**

| 상황 | 기대 동작 |
|---|---|
| `lastIdx === -1` (당월 데이터 없음 / `current_ym` null) | 점선·점 강조·캡션 **전부 미노출**. 현재 렌더와 100% 동일해야 한다. |
| 데이터 포인트가 1개뿐 | 세그먼트가 존재하지 않아 점선이 그려지지 않는다. **점 강조 + 캡션만으로 전달된다** — 정상 동작으로 간주하고 별도 처리하지 않는다. |
| 최신월 직전 달의 avg가 null | `spanGaps: true`라 그 이전 점에서 최신월 점까지 한 구간으로 이어지고, 그 구간이 점선이 된다. 의도한 동작(점선 = "끝점이 최신월인 구간"). |

**x축 라벨에 의존하지 말 것:** `options.scales.x.ticks`가 `autoSkip: true, maxTicksLimit: 7`(`L66-68`)이라 **최신월 눈금이 생략될 수 있다** [확인]. 그래서 §3-3 캡션이 월(`2026.07`)을 **텍스트로 직접 명시**한다.

## 5. 구현 후 검수 체크리스트 (디자인팀이 dev 서버에서 직접 확인)

1. 매매 탭 / 전세 탭 **양쪽 모두**에서 부분월 배지·문단이 뜨는가 (전세 조건부가 아님).
2. 전세 탭에서 부분월 문단 → 전세 모집단 문단 **순서**와 간격이 맞는가.
3. 당월 거래가 없는 지역(예: 소규모 시군구)을 조회했을 때 **배지·점선·캡션이 전부 사라지는가** — `current_ym`이 과거 달로 잡히는 케이스.
4. 375px 폭에서 배지가 라벨을 밀어내 월 표기 `(2026.07)`가 잘리지 않는가.
5. 데스크톱에서 카드 5장 높이가 어긋나지 않는가.
6. 차트 마지막 구간만 점선인가 (중간 구간이 점선이 되면 인덱스 산출 버그).
7. 최신월 점이 흰 속 채움 + 반경 5로 보이는가. 나머지 점 외형이 이전과 같은가.
8. 툴팁 숫자가 이전과 **완전히 동일**하고 접미 문구만 붙는가.
9. **표시 수치 회귀 검증**: 같은 지역·같은 시점에서 변경 전/후 `current_avg`, 변동률 4개, 차액 4개, 차트 각 점 값이 전부 동일한가.
10. 스크린리더(또는 캡션 텍스트만 읽기)로 "어느 달이 점선인지" 알 수 있는가.

## 6. 마스터께 올리는 사항

1. **문구 1건을 기획 A안에서 수정했다** — A안 본문의 "일부 거래가 아직 반영되지 않은 상태이며"가 마스터 제약 2의 금지 예시("아직 덜 반영됐습니다" 류)와 직접 충돌한다. §1-2 표대로 원인 서술만 남기고 결과의 부호를 뺐다. 배지 `신고 진행 중`은 기획 권고 그대로 유지.
