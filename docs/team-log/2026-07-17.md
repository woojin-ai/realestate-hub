- **검증**: 마스터 `npx tsc --noEmit` exit 0 + `npm run build` 성공(/api/data dynamic 등록). **qa-team 독립검증 6 PASS / 1 FAIL** — 회귀안전성(정상지역 콜드조회 old와 동일 결과 코드 논증)·연속성 불변식(중간 failed 시 앞 연속구간까지만 저장)·감소방지(Math.max)·status 다운그레이드(자가치유 정상)·0개월/폴백 엣지 PASS. **FAIL 1건**: `last_deal_ym: ymList[newMonthsCollected-1]`이 `months_collected > 요청 months`일 때(예: 13수집 지역을 months=3 조회) out-of-bounds→undefined(old는 `ymList[length-1]`로 항상 in-bounds였음, 회귀). → **마스터 즉시 수정**: `ymList[Math.min(newMonthsCollected, ymList.length)-1]` 상한 클램프. 재검증 tsc exit 0 + build 성공.
- ⚠️ **잔여 한계(다음 라운드 과제)**: 연속 마킹은 (a)"진행 유실→미수렴"과 (b)"failed:true 갭점프"는 막지만, **(c) throttled-empty(failed:false)로 오는 레이트리밋은 여전히 갭 오염 가능**. 화성 완전 면역엔 **콜드 지역 라이브 호출 상한(정확히는 국토부 하위콜 예산 캡 ~40~48콜/요청)** 이 필요 — 정상 지역 회귀 없이(1코드는 13콜) 화성만 5~6개월/요청으로 제한. molit-api에 콜 카운터 스레딩이 필요해 이번 라운드 범위 밖(신중). 우선 연속 마킹으로 수렴성/부분진행부터 확보.
