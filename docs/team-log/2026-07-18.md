## 라운드 19: 크로스 사이트 내비게이션 위젯 이식 (계산기 허브 → 이 프로젝트)

### 배경
사용자가 계산기 허브(`C:\claude code webside`)에 만든 "크로스 사이트 내비게이션 위젯"(계산기 허브/부동산 실거래가 대시보드/청약레이더/대출모아 4개 자매 사이트를 서로 연결, 데스크톱 우측 고정 레일·모바일 우하단 FAB)을 이 프로젝트에도 이식해달라고 마스터를 통해 지시. 원본 스펙(`design/cross-site-nav-widget-spec.md`, 특히 §13 이식 가이드)과 실제 구현 코드(`components/CrossSiteNav.tsx`, `lib/cross-site-links.ts`)를 직접 Read해 스펙 문서가 아니라 **실제(최신) 코드를 기준**으로 이식(대출모아는 스펙 문서엔 `url:null`로 남아있으나 실제 코드엔 이미 `https://loan-compare-ten.vercel.app`로 배포·`status:"live"` 반영돼 있어 이 값을 그대로 가져옴).

### 이 프로젝트 전용 재계산 — 브레이크포인트 (핵심)
계산기 허브는 가장 넓은 컨테이너가 1024px(`max-w-5xl`)라 레일을 `xl`(1280px)부터 노출했지만, 그 값을 그대로 베끼면 안 되는 이유를 직접 검산:
- `grep -rn "max-w-" app/ components/`로 이 프로젝트의 전체 컨테이너 폭을 재확인 — 가장 넓은 것은 `app/page.tsx`(대시보드 홈)의 `max-w-[1200px]`, 나머지(블로그/약관/소개/문의/개인정보)는 전부 `max-w-3xl`(768px)로 더 좁음. `components/`에서 걸리는 유일한 `max-w-`는 `AptDetailModal.tsx`의 모달 폭(560px, 페이지 컨테이너 아님)이라 계산에서 제외.
- 공식(스펙 §4-1): `V_min = C + 2×(오프셋16 + 레일폭80 + 여백목표24)` = `1200 + 240 = 1440px`.
- `xl`(1280px) 검산: 거터 `(1280-1200)/2=40px` < 레일 점유 `16+80=96px` → **여유 -56px(콘텐츠와 겹침, 불가)**.
- `2xl`(1536px) 검산: 거터 `(1536-1200)/2=168px`, 여유 `168-96=72px`(목표 24px보다 넉넉, 안전).
- → **레일 노출 브레이크포인트를 `xl`이 아니라 `2xl`로 확정**(`hidden ... 2xl:flex` / `2xl:hidden`).

### 그 외 이 프로젝트 전용 재계산/치환
- **디자인 토큰**: 이 프로젝트엔 계산기 허브의 `brand-surface`/`brand-border`/`brand-text-secondary`/`brand-text-disabled`/`brand-bg`/`brand-primary`/`brand-text` 토큰이 없음(`app/globals.css`엔 `--color-brand`/`--color-brand-dark`만 존재). `grep -rn "text-gray-\|border-gray-"`로 이 프로젝트의 실제 관례(raw Tailwind gray 팔레트, 130건)를 확인 후 치환: `brand-primary→brand`, `brand-surface→white`, `brand-border→gray-200`, `brand-text-secondary→gray-500`, `brand-text-disabled→gray-400`, `brand-bg→gray-100`. 추가로 마스터 지시에 없던 `brand-text`(기본 텍스트, 계산기 허브 body 태그 토큰) 사용처를 코드 이식 중 직접 발견해 `text-gray-900`(이 프로젝트 h1/h2 등 기존 heading 관례와 동일, `app/terms/page.tsx` 등에서 확인)로 치환 — 그대로 뒀으면 `text-brand-text`가 정의되지 않은 죽은 클래스가 될 뻔한 부분.
- **`lib/focusRing.ts` 없음**: 새 파일을 만들지 않고 Tier A(`FOCUS_RING_CARD`) 값을 `components/CrossSiteNav.tsx` 안에 `brand-primary→brand` 치환한 인라인 상수로 대체.
- **z-index**: `AptDetailModal.tsx`가 오버레이에 `z-[1000]`(다른 곳은 sticky 테이블 헤더용 `z-10/20/30`)을 씀을 확인 — 위젯의 `z-40`이 이보다 낮아 모달이 열리면 자동으로 덮임(의도된 동작), 충돌 없음 확인.
- **`:has()` FAB 자동숨김 규칙 — 이식 중 직접 발견한 이슈**: 계산기 허브 원본 규칙은 `body:has(main input:focus, ...) .cross-site-fab`로 `<main>` 안쪽에 스코프돼 있었으나, `grep "<main" app/`로 확인한 결과 **이 프로젝트는 app/ 전체에 `<main>` 요소가 단 하나도 없다**(모든 페이지가 `<header>`/`<div>`/`<footer>`를 형제로 배치). 그대로 복사했으면 영구히 매치되지 않는 죽은 규칙이 될 뻔해서, `<main>` 조건을 제거하고 `body:has(input:focus, select:focus, textarea:focus) .cross-site-fab`로 재계산해 포팅.
- **마운트 위치**: `SiteFooter` 공용 컴포넌트가 없어 `app/layout.tsx`의 `<body>{children}</body>` 구조에서 `{children}` 바로 다음에 `<CrossSiteNav currentSiteId="realestate-hub" />` 배치(탭 순서상 본문 우선).
