# 2026-07-14 개발팀 작업 로그

## 목표
Supabase 프로젝트가 아직 생성 중이라, 이번 라운드는 Supabase 없이 국토교통부 실거래가 API를 직접 호출해서
화면에 보여주는 최소기능(MVP)까지 구현(마이그레이션 로드맵 2단계). 아파트 매매/전세만 우선 지원.

## 시작 전 확인
- `.claude/agents/dev-team.md`, `docs/planning/migration-architecture.md`, `docs/design/dashboard.md`,
  `supabase/schema.sql`, 원본 `config.py`/`fetcher.py`/`analyzer.py`를 모두 Read하여 파악.
- `node_modules/next/dist/docs/` 확인 결과 해당 경로 자체가 존재하지 않음(표준 Next.js 16.2.10, App Router).
  커스텀 문서가 없어 표준 App Router 컨벤션(Route Handler, `'use client'` 등)으로 구현.

## 구현 내역
- `lib/regions.ts` — 원본 `config.py`의 `REGION_CODES`, `LAWD_TO_SIGUNGU` 포팅. Python `exec()`로 원본을
  로드해 TS 파싱 결과와 자동 diff — 지역코드 114개(LAWD_TO_SIGUNGU), 구 93개, 시/도 9개 모두 100% 일치 확인.
- `lib/molit-api.ts` — 원본 `fetcher.py`의 아파트 매매/전세 API 호출 로직 포팅(빌라/단독은 다음 라운드).
  서비스키 이중인코딩 방지(URL 문자열에 직접 붙이는 방식) 그대로 유지.
  - **버그 수정**: `fast-xml-parser`가 기본적으로 `resultCode` 같은 숫자형 텍스트("00")를 숫자(0)로
    자동 변환해 앞자리 0이 사라지는 문제 발견 → `parseTagValue: false`로 원본 Python
    `ElementTree.findtext()`와 동일하게 모든 값을 문자열로 유지하도록 수정.
- `lib/analyzer.ts` — 원본 `analyzer.py`의 월별 평균가/변동률(전월/3개월/6개월/1년) 계산과
  오피스텔·도시형 필터 키워드 리스트(`NON_APT_KEYWORDS` 67개, `APT_FORCE_KEYWORDS` 289개) 포팅.
  Python 원본과 자동 diff 스크립트로 대조해 100% 일치 확인(임의 축소 없음).
- `app/api/data/route.ts` — `lawd_cd` + `months`를 받아 국토부 API를 병렬(3개 배치) 호출하고
  `buildSummary`/`buildAptStats` 결과를 JSON으로 반환하는 Route Handler.
- `app/page.tsx` 및 컴포넌트(`RegionSelector`, `BuildingTypeToggle`, `DealTypeTabs`, `SummaryCards`,
  `MonthlyChart`, `DealsTable`) — 디자인팀 구성안(`docs/design/dashboard.md`) 그대로 구현.
  헤더 → 지역선택(+건물유형+조회버튼) → 상태배너 → 현재라벨 → 매매/전세탭 → 요약카드 → 월별차트 →
  목록테이블 순서 유지. 상승/하락/유지 색상(`#e53935`/`#1e88e5`/`#888`) 그대로 적용.
  차트는 `chart.js` + `react-chartjs-2` 신규 설치.
- 이번 라운드는 아파트만 실제 조회 가능(빌라/단독 버튼은 노출하되 "지원 준비 중" 안내로 처리).

## 참고: 동시 작업 이슈
작업 도중 동일 경로에서 다른 프로세스가 같은 파일들을 동시에 수정하는 것을 발견함
(`app/api/deals/route.ts` → `app/api/data/route.ts`로 대체, `lib/types.ts` 필드명 변경 등).
두 버전이 기능적으로 동일한 목표를 구현하고 있어, 최종적으로 안정화된 상태를 기준으로
빌드/린트/실제 API 호출 재검증 후 커밋함. 이후 라운드에서는 동시 세션 여부를 먼저 확인 필요.
